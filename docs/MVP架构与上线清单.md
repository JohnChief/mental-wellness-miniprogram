# MVP 架构与上线清单

## 已确定架构

- 前端：原生微信小程序。
- 后端：Flask，部署到微信云托管服务 `flask-ytaf`。
- 环境：`prod-d6g7im3ft632062b9`。
- 数据库：微信云托管 MySQL，数据库名沿用 `flask_demo`。
- 用户身份：仅信任云托管注入的 `X-WX-OPENID`。
- 登录：云托管注入的微信 OpenID 为注册凭证，昵称和头像可选；手机号在活动报名时由用户填写。
- 小程序请求：`wx.cloud.callContainer`，不依赖临时公网测试域名。
- 签到：管理员操作，暂不允许用户自行远程签到。

## 本轮已完成

- 首页、活动页、活动详情、报名页、我的页面、报名记录与协议页面。
- 加载、空数据、错误、活动已满、重复报名、报名截止等基础状态。
- 活动、用户、报名、配置和管理审计数据模型。
- 活动名额事务锁、防重复报名、管理员签到、取消报名和账号注销。
- 开发期 mock 数据，可在后端部署前预览。
- Admin 数据概览、活动新建/编辑/上下架、报名筛选与签到。
- Admin 用户搜索、身份筛选、排序、单个和批量 VIP 管理。
- Admin Session 登录、CSRF、不可逆签到二次确认、Toast 操作提示和审计日志。
- 受 `ADMIN_TEST_TOOLS_ENABLED` 保护的测试用户生成、取消报名和恢复报名工具。
- 数据库字段、关系和业务约束文档。

## 上线前必须由运营方补充

1. 小程序运营主体全称、统一社会信用代码或个人主体信息。
2. 隐私联系人、客服电话或邮箱。
3. 客服二维码、社群二维码、Logo、活动封面和真实活动资料。
4. 微信公众平台选择的服务类目及对应资质。
5. 隐私保护指引中申报姓名、手机号等处理目的。
6. 对“中医、心理疗愈、体质调理”等名称和宣传文案做类目与资质复核。

## 上线前技术检查

- 设置高强度 `SECRET_KEY` 与 `ADMIN_API_KEY`，不得提交到 Git。
- 在云托管环境变量中配置 `WECHAT_APP_ID`、`WECHAT_APP_SECRET`，不得写入前端。
- 后端部署成功后关闭 `SEED_SAMPLE_DATA`。
- 确认前端 `useMock` 为 `false`。
- 验证云托管 OpenID 请求头和手动手机号报名流程。
- 管理端正式开放前配置密码哈希、登录限速和安全 Cookie。
- 正式运营前关闭 `ADMIN_TEST_TOOLS_ENABLED`。
- 配置数据库备份、日志脱敏、最小权限和数据保留期限。
- 真机测试弱网、重复点击、活动满员、取消报名和账号注销。
